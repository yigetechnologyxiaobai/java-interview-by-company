### 一面真题

#### Q1: String 的长度有限制吗？

**参考答案**：
- **编译期**：String 常量池中字符串最大长度为 65534（UTF-8 编码）
- **运行期**：`String` 内部用 `char[]` 存储，最大长度为 `Integer.MAX_VALUE`（约 21 亿）

#### Q2: 三个线程各执行 50 次 i++，如何保证结果是 150？

**参考答案**：

```java
// 方案 1：synchronized
private static int count = 0;
public synchronized static void increment() {
    count++;
}

// 方案 2：AtomicInteger
private static AtomicInteger count = new AtomicInteger(0);
public static void increment() {
    count.incrementAndGet();
}

// 方案 3：ReentrantLock
private static int count = 0;
private static ReentrantLock lock = new ReentrantLock();
public static void increment() {
    lock.lock();
    try {
        count++;
    } finally {
        lock.unlock();
    }
}
```

#### Q3: 怎么解决消息队列丢数据？

**参考答案**：

| 阶段 | 问题 | 解决方案 |
|------|------|---------|
| 生产者 | 发送失败 | ACK 确认 + 重试 |
| Broker | 宕机丢失 | 持久化 + 多副本 |
| 消费者 | 处理失败 | 手动提交 offset |

**Kafka 配置**：
```properties
# 生产者
acks=all                          # 等待所有副本确认
retries=3                         # 重试次数

# Broker
min.insync.replicas=2             # 最小同步副本数
unclean.leader.election.enable=false  # 禁止非同步副本成为 leader

# 消费者
enable.auto.commit=false          # 手动提交 offset
```

#### Q4: 手撕 - 合并 K 个升序链表

**参考解答**：
```java
public ListNode mergeKLists(ListNode[] lists) {
    if (lists == null || lists.length == 0) return null;
    
    PriorityQueue<ListNode> heap = new PriorityQueue<>((a, b) -> a.val - b.val);
    
    for (ListNode node : lists) {
        if (node != null) heap.offer(node);
    }
    
    ListNode dummy = new ListNode(0);
    ListNode cur = dummy;
    
    while (!heap.isEmpty()) {
        ListNode node = heap.poll();
        cur.next = node;
        cur = cur.next;
        if (node.next != null) heap.offer(node.next);
    }
    
    return dummy.next;
}
```

### 二面真题

#### Q1: MySQL 慢查询优化方案？

**参考答案**：

1. **分析慢查询**：
   - 开启慢查询日志
   - 使用 `EXPLAIN` 分析执行计划

2. **优化方案**：
   - 添加合适的索引
   - 避免索引失效（函数、类型转换、!=、or）
   - 只查询需要的字段（避免 SELECT *）
   - 分页优化（覆盖索引 + 子查询）

#### Q2: 面向对象 vs 面向过程？

**参考答案**：

| 特性 | 面向过程 | 面向对象 |
|------|---------|---------|
| 核心思想 | 过程、步骤 | 对象、交互 |
| 数据安全 | 数据暴露 | 封装保护 |
| 复用性 | 函数复用 | 继承、多态 |
| 适用场景 | 简单、性能敏感 | 复杂、可扩展 |

### 三面真题

#### Q1: 怎么自己设计一个消息队列？

**参考答案**：

**核心组件**：
1. **Broker**：消息存储节点
2. **NameServer**：路由注册中心
3. **Producer**：消息生产者
4. **Consumer**：消息消费者

**关键设计**：
- **存储**：顺序写 + 零拷贝
- **高可用**：主从复制 + 故障转移
- **高性能**：批量发送 + 异步刷盘
- **可靠性**：ACK 确认 + 消息重试

---

#### Q2: 缓存污染是什么？

**参考答案**：

**定义**：缓存中存储了不常用或无效数据，挤占了热点数据的内存。

**场景**：
- 一次查询大量数据
- 批量导入数据时缓存全部数据

**解决方案**：
1. **设置合理的过期时间**
2. **使用 LRU/LFU 淘汰策略**
3. **禁止批量加载到缓存**
4. **使用布隆过滤器过滤无效查询**
