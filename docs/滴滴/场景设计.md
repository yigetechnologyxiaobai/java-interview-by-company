## 业务场景题


### Q14: 派单系统设计？

**参考答案**：

**核心功能**：
- 订单创建
- 司机匹配
- 订单派发
- 状态更新

**匹配算法**：
- 距离优先
- 时间优先
- 综合评分

**架构设计**：
```
用户 → 网关 → 订单服务 → 匹配服务 → 司机服务
                      ↓
                    Redis（地理围栏）
```

**地理围栏**：
- Redis GEO
- 计算司机和乘客距离
- 筛选附近司机

**追问**：
- 如何保证派单公平？
- 如何处理高峰期？

### Q15: 订单系统设计？

**参考答案**：

**核心模块**：
1. 订单创建：幂等设计
2. 订单支付：状态机
3. 订单履约：状态流转
4. 订单取消：库存释放

**状态机**：
```
待接单 → 已接单 → 进行中 → 已完成
   ↓                  ↓
  取消               取消
```

**关键设计**：
- 订单号生成（Snowflake）
- 分布式事务
- 分库分表
- 延迟队列处理超时

**追问**：
- 如何保证订单号唯一？
- 如何处理订单超时？

### Q16: 秒杀系统设计？

**参考答案**：

**架构分层**：
```
用户 → CDN → 网关 → 应用服务 → 缓存 → MQ → 数据库
```

**核心设计**：
1. 前端限流
2. 网关限流
3. Redis 预扣库存
4. MQ 异步下单
5. 防超卖

**追问**：
- 如何防止超卖？
- 如何防止重复下单？
