## 分布式高频题


### Q1: 分布式锁实现？

**参考答案**：

**Redis 实现**：
```redis
# 加锁
SET lock:key value NX PX 30000

# 解锁（Lua 脚本保证原子性）
if redis.call("GET", KEYS[1]) == ARGV[1] then
    return redis.call("DEL", KEYS[1])
else
    return 0
end
```

**Zookeeper 实现**：
- 临时顺序节点
- EPHEMERAL_SEQUENTIAL
- Watch 机制

**对比**：
| 方案 | 性能 | 可靠性 | 适用场景 |
|------|------|--------|---------|
| Redis | 高 | 中 | 性能优先 |
| Zookeeper | 中 | 高 | 可靠性优先 |

**Redisson 增强**：
- 可重入锁
- 看门狗自动续期
- 红锁（Redlock）

**追问**：
- 锁过期了但业务没执行完怎么办？
- 主从切换导致锁丢失怎么办？

### Q2: 分布式事务？

**参考答案**：

| 方案 | 原理 | 场景 |
|------|------|------|
| 2PC | 准备 + 提交 | 强一致 |
| TCC | Try-Confirm-Cancel | 高并发 |
| 本地消息表 | 消息 + 定时任务 | 最终一致 |
| 事务消息 | RocketMQ 半消息 | 最终一致 |
| Seata AT | 全局锁 + Undo Log | 简单场景 |

**TCC 流程**：
1. Try：预留资源
2. Confirm：确认提交
3. Cancel：回滚释放

**TCC 问题**：
- **空回滚**：Try 未执行，Cancel 被调用
- **悬挂**：Cancel 先于 Try 执行
- **幂等**：Confirm/Cancel 需要幂等

**追问**：
- TCC 和 2PC 区别？
- Seata AT 模式原理？

### Q3: 分布式 ID 生成？

**参考答案**：

| 方案 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| UUID | 随机生成 | 简单 | 无序 |
| 数据库自增 | 自增 ID | 简单 | 单点 |
| Redis | INCR | 高性能 | 依赖 Redis |
| Snowflake | 时间戳 + 机器 ID + 序列号 | 分布式 | 依赖时钟 |
| Leaf | Snowflake + 数据库 | 综合 | 复杂 |

**Snowflake 结构**：
```
0 - 41bit 时间戳 - 10bit 机器 ID - 12bit 序列号
```

**时钟回拨问题**：
- 等待时间追上
- 使用备用机器 ID
- 抛异常拒绝

**追问**：
- Snowflake 如何保证唯一？
- 如何处理时钟回拨？

### Q4: 分布式 Session？

**参考答案**：

**方案对比**：

| 方案 | 描述 | 优缺点 |
|------|------|--------|
| 粘性 Session | 绑定用户到服务器 | 简单，但不均衡 |
| Session 复制 | 服务器间同步 Session | 复杂，性能差 |
| 集中式存储 | Redis 存储 Session | 推荐 |

**Redis 实现**：
```java
// 登录时
redis.set("session:" + token, userJson, 3600);

// 请求时
String userJson = redis.get("session:" + token);
```

**JWT 方案**：
- 无状态
- 不需要存储
- 但无法主动失效

**追问**：
- JWT 和 Session 区别？
- 如何实现单点登录？
