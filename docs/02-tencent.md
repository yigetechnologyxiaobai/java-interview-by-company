## 二、计算机网络高频题

### Q1: TCP 三次握手？

**参考答案**：

```
客户端                    服务端
   │                        │
   │───SYN=1, seq=x────────►│  第一次握手
   │                        │
   │◄──SYN=1, ACK=1, ack=x+1│  第二次握手
   │    seq=y               │
   │                        │
   │───ACK=1, ack=y+1───────►│  第三次握手
   │                        │
```

**原因**：防止已失效的连接请求突然又传送到服务端

**追问**：
- 为什么不是两次？（防止历史连接）
- SYN 攻击是什么？

### Q2: TCP 四次挥手？

**参考答案**：

```
客户端                    服务端
   │                        │
   │───FIN=1, seq=u────────►│  第一次挥手
   │                        │
   │◄──ACK=1, ack=u+1───────│  第二次挥手
   │                        │
   │◄──FIN=1, ACK=1, seq=w──│  第三次挥手
   │                        │
   │───ACK=1, ack=w+1───────►│  第四次挥手
   │                        │
```

**TIME_WAIT 状态**：
- 持续 2MSL
- 确保最后一个 ACK 到达
- 让旧连接的包消失

**追问**：
- 为什么客户端要等 2MSL？
- TIME_WAIT 过多怎么办？

### Q3: TCP 和 UDP 区别？

| TCP | UDP |
|-----|-----|
| 面向连接 | 无连接 |
| 可靠传输 | 不可靠 |
| 有序 | 无序 |
| 流量控制 | 无 |
| 拥塞控制 | 无 |
| 传输效率低 | 传输效率高 |

**追问**：
- 什么时候用 TCP？什么时候用 UDP？
- TCP 如何保证可靠传输？

### Q4: HTTP 和 HTTPS 区别？

| HTTP | HTTPS |
|------|-------|
| 明文传输 | SSL 加密 |
| 默认 80 端口 | 默认 443 端口 |
| 无证书 | 需要证书 |

**HTTPS 握手过程**：
1. 客户端发送支持的加密算法
2. 服务端返回证书和公钥
3. 客户端验证证书，生成对称密钥
4. 用公钥加密对称密钥发送
5. 服务端用私钥解密得到对称密钥

**追问**：
- 对称加密和非对称加密的区别？
- 为什么 HTTPS 混合使用两种加密？

### Q5: 输入 URL 到页面展示的过程？

**参考答案**：

1. **DNS 解析**：域名 → IP
2. **TCP 连接**：三次握手
3. **HTTP 请求**：发送请求
4. **服务器处理**：返回响应
5. **浏览器渲染**：解析 HTML/CSS/JS

**追问**：
- DNS 解析过程？
- 浏览器缓存机制？
- 重绘和回流？

- 线程同步方式？

### Q7: 死锁条件？

**四个必要条件**：

1. **互斥**：资源只能被一个进程使用
2. **请求保持**：持有资源同时请求其他资源
3. **不剥夺**：资源不能被强制抢占
4. **循环等待**：进程间形成循环等待关系

**解决方法**：
- 预防：破坏四个条件之一
- 避免：银行家算法
- 检测：资源分配图
- 解除：剥夺资源、终止进程

### Q8: 进程调度算法？

1. **先来先服务（FCFS）**
2. **短作业优先（SJF）**
3. **时间片轮转**
4. **优先级调度**
5. **多级反馈队列**

### Q9: 内存管理方式？

1. **分页**：固定大小，减少碎片
2. **分段**：逻辑单位，便于共享
3. **段页式**：结合两者

**虚拟内存**：
- 逻辑上扩充内存
- 页面置换算法：LRU、FIFO、Clock

## 四、Java 基础高频题

### Q10: HashMap 源码分析？

**JDK 1.8 结构**：数组 + 链表 + 红黑树

**put 流程**：
1. 计算 hash
2. 定位桶
3. 桶为空 → 直接插入
4. 桶不为空 → 遍历链表/红黑树
5. 相同 key → 覆盖 value
6. 不同 key → 尾插法插入

**扩容**：size > capacity * loadFactor，扩容 2 倍

**追问**：
- JDK 1.7 和 1.8 区别？（头插法 vs 尾插法）
- 为什么线程不安全？

### Q11: ConcurrentHashMap 如何保证线程安全？

**JDK 1.8**：
- 桶为空 → CAS 插入
- 桶不为空 → synchronized 锁头节点

**追问**：
- 为什么放弃分段锁？
- 和 Hashtable 区别？

### Q12: ThreadLocal 原理？

每个 Thread 维护一个 ThreadLocalMap，key 是 ThreadLocal（弱引用），value 是值（强引用）。

**内存泄漏**：key 被 GC 后变 null，value 无法回收，需要手动 remove()

### Q16: 缓存穿透、击穿、雪崩？

| 问题 | 原因 | 解决 |
|------|------|------|
| 穿透 | 查询不存在的数据 | 布隆过滤器、缓存空值 |
| 击穿 | 热点 key 过期 | 永不过期、互斥锁 |
| 雪崩 | 大量 key 同时过期 | 随机过期时间、多级缓存 |
