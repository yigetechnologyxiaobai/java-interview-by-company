#### Q3: 多线程的运用场景？（CountDownLatch）

**参考答案**：

**CountDownLatch 场景**：
- **并行任务等待**：多个子任务并行执行，主线程等待所有子任务完成
- **服务启动**：等待所有依赖服务启动完成
- **批量处理**：等待批量任务处理完成

**代码示例**：
```java
// 并行处理示例
CountDownLatch latch = new CountDownLatch(5);

for (int i = 0; i < 5; i++) {
    new Thread(() -> {
        try {
            // 执行任务
            processTask();
        } finally {
            latch.countDown();  // 计数器减1
        }
    }).start();
}

latch.await();  // 主线程等待，计数器归0时继续
System.out.println("所有任务完成");
```

**其他多线程场景**：
- **CompletableFuture**：异步编排
- **Semaphore**：限流
- **CyclicBarrier**：循环等待

#### Q4: 排查过什么线上问题？

**参考答案**：

**常见线上问题排查**：

1. **CPU 高**：
   ```bash
   top -H -p pid  # 找到高CPU线程
   jstack pid     # 查看线程堆栈
   ```

2. **内存泄漏**：
   ```bash
   jmap -histo pid | head 20  # 查看对象数量
   jmap -dump:format=b,file=heap.hprof pid  # dump分析
   ```

3. **线程死锁**：
   ```bash
   jstack pid | grep -A 10 "deadlock"
   ```

4. **慢SQL**：
   ```bash
   show processlist;  # 查看执行中的SQL
   explain select...  # 分析执行计划
   ```

#### Q6: 介绍一下 Netty

**参考答案**：

**Netty 核心组件**：

| 组件 | 作用 |
|------|------|
| Bootstrap | 启动引导类 |
| Channel | 网络通道 |
| EventLoop | 事件循环（IO线程） |
| ChannelHandler | 业务处理器 |
| ChannelPipeline | Handler 链 |
| ByteBuf | 字节缓冲区 |

**Netty 模型**：
- Reactor 模型
- 主从 Reactor 多线程模型

**优势**：
- 高性能、高并发
- 零拷贝（ByteBuf）
- 异步非阻塞

**应用场景**：
- RPC 框架（Dubbo）
- 消息推送
- 游戏服务器

#### Q7: JVM 调优过吗？

**参考答案**：

**调优指标**：
- GC 频率
- GC 耗时
- 响应时间
- 吞吐量

**常用参数**：
```bash
# 堆大小
-Xms2g -Xmx2g

# 新生代大小
-Xmn1g

# 元空间
-XX:MetaspaceSize=256m -XX:MaxMetaspaceSize=512m

# GC 日志
-Xlog:gc*:file=gc.log

# G1 参数
-XX:+UseG1GC
-XX:MaxGCPauseMillis=200
```

**调优思路**：
1. 分析 GC 日志
2. 找出频繁 GC 的原因
3. 调整堆大小、分代比例
4. 选择合适的 GC 算法
