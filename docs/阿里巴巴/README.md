## 七、项目深挖


**常见追问**：

1. 这个项目的难点是什么？
2. 为什么选择这个技术方案？
3. 有没有考虑其他方案？对比分析？
4. 遇到过什么问题？怎么解决的？
5. 如果让你重新设计，会怎么改进？
6. 系统的瓶颈在哪里？如何优化？

**回答框架**：

```
项目背景 → 问题分析 → 方案对比 → 最终选择 → 遇到的问题 → 解决方案 → 效果评估
```

---


## 八、面试技巧


1. **不要只回答是什么，要回答为什么**
2. **准备好项目的每个细节**
3. **不懂就说不懂，不要乱编**
4. **准备好反问环节**
5. **展示学习能力**：最近在看什么书/文章？

**题目**：智能安全邮箱设计

#### 题目目标
在本地环境实现一个"可用 + 可测 + 有安全设计"的邮箱系统，包含服务端与客户端，并引入基础智能能力。

#### 总体约束
- 使用指定技术栈
- 满足功能和非功能需求

#### 必做功能

**1. 服务器管理**
- 实现服务端进程，负责接收、存储、分发邮件

**2. 客户端管理**
- 实现客户端进程，负责客户使用邮箱、查看发送邮件
- 支持基本邮件操作

#### 算法增强功能（至少实现 2 项）
- 垃圾邮件检测
- 钓鱼识别
- 智能分类
- 邮件摘要生成

#### 安全与稳定性（必做）
- 身份认证
- 数据加密
- 异常处理
- 日志记录

#### 测试与验收要求
- 单元测试
- 集成测试
- 性能测试

#### 加分项
- 分布式部署
- 高可用设计
- AI 能力集成
