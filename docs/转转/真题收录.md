### 一面真题

#### Q1: 走了索引还是慢，怎么排查？

**参考答案**：

**排查步骤**：
1. **EXPLAIN 分析执行计划**：
   - 查看是否真正使用了索引
   - 检查扫描行数（rows）
   - 是否有 Using filesort、Using temporary

2. **检查索引选择性**：
   - 选择性低（如性别）的索引可能不生效
   - 通过 `SHOW INDEX FROM table` 查看索引基数

3. **回表次数过多**：
   - 非聚簇索引需要回表查询
   - 考虑使用覆盖索引

4. **索引列上有计算/函数**：
   ```sql
   -- 索引失效
   WHERE YEAR(create_time) = 2026
   -- 优化
   WHERE create_time >= '2026-01-01' AND create_time < '2027-01-01'
   ```

5. **数据量问题**：
   - 单表数据量过大（> 1000万）
   - 考虑分表

#### Q2: ZK（ZooKeeper）的角色有哪些？如何理解强一致性？

**参考答案**：

**ZK 角色**：

| 角色 | 作用 |
|------|------|
| Leader | 处理写请求，协调 Follower |
| Follower | 处理读请求，参与选举投票 |
| Observer | 处理读请求，不参与投票（提升读性能） |

**强一致性理解**：

ZK 保证**最终一致性**而非严格的强一致性：

1. **写请求**：必须经过 Leader，多数节点确认后返回
2. **读请求**：可以从任意 Follower/Observer 读取（可能读到旧数据）
3. **同步机制**：Leader 将变更同步给所有 Follower

**ZAB 协议**：
- **崩溃恢复**：Leader 崩溃后重新选举
- **消息广播**：Leader 广播变更给 Follower

#### Q3: Redis 如何确保加锁一定成功？

**参考答案**：

**原生 Redis 分布式锁的问题**：

| 问题 | 说明 |
|------|------|
| **锁误删** | A 的锁过期，B 获取锁，A 释放了 B 的锁 |
| **不可重入** | 同一线程无法多次获取同一锁 |
| **无等待机制** | 获取失败直接返回，无阻塞等待 |
| **无看门狗** | 业务执行超过过期时间，锁被自动释放 |

**Redisson 解决方案**：

```java
// 1. 锁误删：Lua 脚本校验
if redis.call("get", KEYS[1]) == ARGV[1] then
    return redis.call("del", KEYS[1])
end

// 2. 可重入：hash 结构存储线程ID和计数
// lock_key = {thread_id: count}

// 3. 等待机制：pub/sub 订阅锁释放通知

// 4. 看门狗：后台线程自动续期
```

**看门狗续签机制**：
- 锁默认过期时间 30s
- 后台线程每隔 10s（过期时间/3）检查
- 如果锁还被当前线程持有，续期 30s

#### Q6: 三色标记算法是什么？

**参考答案**：

**三色标记算法**是 CMS 和 G1 垃圾收集器使用的并发标记算法。

**三种颜色**：
- **白色**：未被访问的对象（可能被回收）
- **灰色**：被访问但子对象未全部访问
- **黑色**：被访问且子对象全部访问（不会被回收）

**漏标问题（对象消失）**：
- 灰色对象指向白色对象的引用被删除
- 同时黑色对象指向白色对象
- 白色对象被错误回收！

**解决方案**：
- **CMS**：增量更新（记录新增引用，重新扫描）
- **G1**：SATB（Snapshot-At-The-Beginning，开始时快照）

#### Q1: 分片查询怎么防止 SQL 注入？

**参考答案**：

**分片查询 SQL 注入风险**：
```java
// 危险写法
String shardKey = request.getParameter("shardKey");
String sql = "SELECT * FROM user_" + shardKey + " WHERE id = ?";
// shardKey = "1; DROP TABLE user_1; --" → SQL注入
```

**防护措施**：

1. **白名单校验**：
```java
Set<String> validTables = Set.of("user_0", "user_1", "user_2");
if (!validTables.contains("user_" + shardKey)) {
    throw new IllegalArgumentException("Invalid shard key");
}
```

2. **正则校验**：
```java
if (!shardKey.matches("\\d+")) {
    throw new IllegalArgumentException("Invalid shard key format");
}
```

3. **使用 ShardingSphere 等中间件**：中间件自动路由

#### Q4: 间隙锁导致的死锁怎么修复？

**参考答案**：

**间隙锁死锁场景**：

```sql
-- 表中有 id = 1, 5, 10

-- 事务 A
SELECT * FROM user WHERE id = 3 FOR UPDATE;  -- 加间隙锁 (1, 5)

-- 事务 B
SELECT * FROM user WHERE id = 7 FOR UPDATE;  -- 加间隙锁 (5, 10)
INSERT INTO user (id, name) VALUES (4, 'test');  -- 等待 A 的间隙锁

-- 事务 A
INSERT INTO user (id, name) VALUES (6, 'test');  -- 等待 B 的间隙锁
-- 死锁！
```

**解决方案**：

1. **避免间隙锁**：使用唯一索引查询存在的记录
2. **降低隔离级别**：读提交隔离级别无间隙锁
3. **加锁顺序一致**：统一按 id 升序加锁

### 手撕算法

#### 一面算法：字符串A/B移动最小步数

**题目**：一个字符串，只包含A、B两个字符，求把所有A移动到左边，所有B移动到右边的最小步数。

**参考答案**：

```java
public int minSteps(String s) {
    int countB = 0, steps = 0;
    for (char c : s.toCharArray()) {
        if (c == 'B') {
            countB++;
        } else {
            steps += countB;  // 每个A需要跨过前面所有B
        }
    }
    return steps;
}
```

#### 二面算法：无重复字符最长子串

**题目**：给定一个字符串 s ，找出其中不含有重复字符的最长子串的长度。

**参考答案**：

```java
public int lengthOfLongestSubstring(String s) {
    Map<Character, Integer> map = new HashMap<>();
    int maxLen = 0, left = 0;
    
    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        if (map.containsKey(c)) {
            left = Math.max(left, map.get(c) + 1);
        }
        map.put(c, right);
        maxLen = Math.max(maxLen, right - left + 1);
    }
    
    return maxLen;
}
```
