### Q1: 字符串只包含 A、B，把所有 A 移到左边，所有 B 移到右边的最小步数？

**参考答案**：

**题目分析**：
- 每次可以交换任意两个字符的位置
- 最小步数 = min(A 的数量, B 的数量) 中需要移动的次数

**解法1：双指针**：
```java
public int minSteps(String s) {
    char[] arr = s.toCharArray();
    int left = 0, right = arr.length - 1;
    int steps = 0;
    
    while (left < right) {
        // 找左边第一个 B
        while (left < right && arr[left] == 'A') left++;
        // 找右边第一个 A
        while (left < right && arr[right] == 'B') right--;
        
        if (left < right) {
            // 交换
            arr[left] = 'A';
            arr[right] = 'B';
            steps++;
            left++;
            right--;
        }
    }
    return steps;
}
```

**解法2：计数法**：
```java
public int minSteps(String s) {
    int countA = 0;
    for (char c : s.toCharArray()) {
        if (c == 'A') countA++;
    }
    
    // 检查前 countA 个位置有多少个 B
    int wrongB = 0;
    for (int i = 0; i < countA; i++) {
        if (s.charAt(i) == 'B') wrongB++;
    }
    return wrongB;
}
```

### Q2: 最长不重复字符子串长度？

**参考答案**：

**题目**：给定字符串 s，找出不含重复字符的最长子串长度。

**解法：滑动窗口**：
```java
public int lengthOfLongestSubstring(String s) {
    Map<Character, Integer> map = new HashMap<>();
    int maxLen = 0, left = 0;
    
    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        if (map.containsKey(c) && map.get(c) >= left) {
            left = map.get(c) + 1;  // 移动左边界
        }
        map.put(c, right);
        maxLen = Math.max(maxLen, right - left + 1);
    }
    return maxLen;
}
```

**追问**：
- 时间复杂度？O(n)
- 如果字符串只包含小写字母怎么办？
  - 可以用数组代替 HashMap，效率更高

### Q3: 走了索引还是慢，怎么排查？

**参考答案**：

**排查步骤**：

1. **确认索引是否生效**：
```sql
EXPLAIN SELECT * FROM user WHERE name = 'Tom';
-- 查看 type、key、key_len 字段
```

2. **可能的原因**：

| 原因 | 说明 | 解决方案 |
|------|------|---------|
| 回表查询 | 非聚簇索引需要回表 | 覆盖索引 |
| 索引选择性低 | 数据重复率高 | 优化索引 |
| 扫描行数多 | 范围查询或模糊查询 | 限制结果 |
| 锁等待 | 其他事务持有锁 | 检查锁 |
| 统计信息不准 | 优化器判断错误 | ANALYZE TABLE |

3. **解决方案**：
```sql
-- 覆盖索引
CREATE INDEX idx_name_age ON user(name, age);
SELECT name, age FROM user WHERE name = 'Tom';  -- 不回表

-- 强制使用索引
SELECT * FROM user FORCE INDEX(idx_name) WHERE name = 'Tom';
```

### Q4: ZK 的角色有哪些？如何理解强一致性？

**参考答案**：

**ZK 角色**：

| 角色 | 作用 |
|------|------|
| Leader | 处理写请求，协调 Follower |
| Follower | 处理读请求，参与选举投票 |
| Observer | 处理读请求，不参与投票（提升读性能） |

**强一致性理解**：

ZK 保证**最终一致性**而非严格的强一致性：

1. **写请求**：必须经过 Leader，多数节点确认后返回
2. **读请求**：可以从任意 Follower/Observer 读取（可能读到旧数据）
3. **同步机制**：Leader 将变更同步给所有 Follower

**强一致性读**：
```java
zk.sync("/path", null, null);  // 同步到最新状态
byte[] freshData = zk.getData("/path", false, null);
```

### Q5: Redis 如何确保加锁一定成功？

**参考答案**：

**Redis 加锁要点**：

```java
// SET NX EX 原子操作
SET lock_key unique_value NX EX 30

// NX：不存在才设置（互斥）
// EX：设置过期时间（防止死锁）
```

**完整实现**：
```java
public boolean tryLock(String key, String requestId, int expireTime) {
    String result = redis.set(key, requestId, "NX", "EX", expireTime);
    return "OK".equals(result);
}

public boolean releaseLock(String key, String requestId) {
    // Lua 脚本保证原子性
    String script = 
        "if redis.call('get', KEYS[1]) == ARGV[1] then " +
        "return redis.call('del', KEYS[1]) " +
        "else return 0 end";
    return redis.eval(script, Arrays.asList(key), Arrays.asList(requestId));
}
```

**注意**：
- 使用唯一 requestId 防止误删其他线程的锁
- 释放锁用 Lua 脚本保证原子性

### Q6: 线程池定义规范有什么经验？

**参考答案**：

**线程池命名规范**：
```java
// 自定义 ThreadFactory
ThreadFactory factory = new ThreadFactoryBuilder()
    .setNameFormat("order-process-%d")  // 线程名前缀
    .setDaemon(false)                   // 非守护线程
    .setPriority(Thread.NORM_PRIORITY)  // 正常优先级
    .setUncaughtExceptionHandler((t, e) -> {
        log.error("Thread {} error", t.getName(), e);
    })
    .build();
```

**参数设置经验**：

| 参数 | 建议 |
|------|------|
| 核心线程数 | CPU 密集：核心数+1；IO 密集：核心数×2 |
| 最大线程数 | 核心线程数 × 2 ~ 4 |
| 队列容量 | 根据业务峰值设置，避免过大导致 OOM |
| 拒绝策略 | CallerRunsPolicy（降级执行） |

**监控**：
```java
// 定期监控线程池状态
executor.getActiveCount();    // 活跃线程数
executor.getQueueSize();      // 队列大小
executor.getCompletedTaskCount();  // 完成任务数
```

### Q7: JVM CMS 和 G1 的特点？

### Q8: Redisson 看门狗怎么续签？

**参考答案**：

**看门狗原理**：
- 锁默认过期时间 30s
- 后台线程每隔 10s（过期时间/3）检查
- 如果锁还被当前线程持有，续期 30s
- 业务执行完毕，主动释放锁，看门狗停止

**代码示例**：
```java
RLock lock = redisson.getLock("myLock");
lock.lock();  // 启动看门狗
try {
    // 业务逻辑（可能超过 30s）
} finally {
    lock.unlock();  // 停止看门狗
}
```

**参考答案**：
