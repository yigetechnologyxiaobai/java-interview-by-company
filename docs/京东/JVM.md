## 三、JVM 高频题


### Q10: JVM 内存结构？

**参考答案**：

**运行时数据区**：
1. **程序计数器**：当前线程执行的字节码行号
2. **虚拟机栈**：方法调用的栈帧
3. **本地方法栈**：Native 方法服务
4. **堆**：对象实例存储，GC 主要区域
5. **方法区**：类信息、常量、静态变量

**堆内存划分**：
- 新生代（1/3）：Eden + Survivor0 + Survivor1
- 老年代（2/3）

**追问**：
- 堆和栈的区别？
- 方法区在不同 JDK 版本的变化？

### Q11: 垃圾收集算法？

**参考答案**：

| 算法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 标记-清除 | 标记后清除 | 简单 | 碎片 |
| 标记-整理 | 标记后整理 | 无碎片 | 移动成本 |
| 复制算法 | 复制存活对象 | 高效 | 空间浪费 |
| 分代收集 | 按分代选算法 | 综合 | 复杂 |

**分代收集**：
- 新生代：复制算法（存活率低）
- 老年代：标记-整理（存活率高）

**追问**：
- 为什么新生代用复制算法？
- CMS 为什么用标记-清除？

### Q12: 垃圾收集器？

**参考答案**：

| 收集器 | 类型 | 特点 |
|--------|------|------|
| Serial | 单线程 | 简单，适合客户端 |
| Parallel | 并行 | 吞吐优先 |
| CMS | 并发 | 低延迟，有碎片 |
| G1 | 分代 | 可预测停顿，无碎片 |
| ZGC | 并发 | 超大内存，低延迟 |

**CMS 四个阶段**：
1. 初始标记（STW）
2. 并发标记
3. 重新标记（STW）
4. 并发清除

**G1 特点**：
- Region 分区
- 可预测停顿
- 无碎片

**追问**：
- CMS 和 G1 的区别？
- 如何选择垃圾收集器？

### Q13: 类加载过程？

**参考答案**：

1. **加载**：读取 Class 文件，创建 Class 对象
2. **验证**：确保 Class 文件合法
3. **准备**：为静态变量分配内存，设初始值
4. **解析**：符号引用转直接引用
5. **初始化**：执行 `<clinit>` 方法

**双亲委派模型**：
- 自下而上委托
- 自上而下加载

**打破双亲委派**：
- 自定义类加载器
- SPI 机制

**追问**：
- 为什么要双亲委派？
- Tomcat 如何打破双亲委派？
