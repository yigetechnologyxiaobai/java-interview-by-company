## MySQL 高频题


### Q14: 索引为什么用 B+ 树？

**参考答案**：

**B+ 树特点**：
1. 树高度低：3-4 层可存千万级数据
2. 范围查询友好：叶子节点链表
3. 磁盘利用率高：非叶子节点只存键值

**与 B 树区别**：
- B 树：每个节点都存数据
- B+ 树：只有叶子节点存数据

**与 Hash 索引区别**：
- Hash：等值查询快，不支持范围
- B+：支持范围、排序

### Q15: 索引失效场景？

**参考答案**：

1. **LIKE 以 % 开头**：
   ```sql
   SELECT * FROM t WHERE name LIKE '%abc';  -- 不走索引
   SELECT * FROM t WHERE name LIKE 'abc%';   -- 走索引
   ```

2. **对索引列做运算**：
   ```sql
   SELECT * FROM t WHERE YEAR(create_time) = 2024;  -- 不走索引
   ```

3. **类型隐式转换**：
   ```sql
   SELECT * FROM t WHERE phone = 13800000000;  -- phone 是字符串，不走索引
   ```

4. **使用函数**：
   ```sql
   SELECT * FROM t WHERE LOWER(name) = 'abc';  -- 不走索引
   ```

5. **OR 条件**：
   ```sql
   SELECT * FROM t WHERE a = 1 OR b = 2;  -- b 无索引，全表扫描
   ```

6. **不等于（!=、<>）**

7. **IS NOT NULL**

### Q16: 事务隔离级别？

**参考答案**：

| 隔离级别 | 脏读 | 不可重复读 | 幻读 |
|---------|------|-----------|------|
| READ UNCOMMITTED | ✅ | ✅ | ✅ |
| READ COMMITTED | ❌ | ✅ | ✅ |
| REPEATABLE READ | ❌ | ❌ | ✅ |
| SERIALIZABLE | ❌ | ❌ | ❌ |

**MySQL 默认**：REPEATABLE READ

**MVCC 原理**：
- 隐藏字段：DB_TRX_ID、DB_ROLL_PTR
- Undo Log：版本链
- Read View：可见性判断

**追问**：

### Q17: 主从复制原理？

**参考答案**：

**复制流程**：
1. Master 写入 Binlog
2. Slave IO 线程读取 Binlog，写入 Relay Log
3. Slave SQL 线程重放 Relay Log

**主从延迟原因**：
1. 单线程重放
2. 大事务
3. 网络延迟
4. 硬件性能差

**解决方案**：
1. 并行复制
2. 半同步复制
3. 读写分离优化

**追问**：
- 如何保证主从一致性？
- 半同步复制原理？

## Redis 高频题


### Q18: Redis 为什么快？

**参考答案**：

1. **内存操作**：数据在内存中
2. **单线程**：无锁竞争
3. **IO 多路复用**：epoll
4. **高效数据结构**：SDS、跳表、压缩列表

**追问**：
- 为什么用单线程？
- Redis 6.0 为什么引入多线程？

### Q19: Redis 数据结构？

**参考答案**：

| 类型 | 底层实现 | 场景 |
|------|---------|------|
| String | SDS | 缓存、计数器 |
| Hash | 哈希表 + 压缩列表 | 对象 |
| List | 压缩列表 + 双向链表 | 队列 |
| Set | 哈希表 + 整数集合 | 标签 |
| ZSet | 跳表 + 哈希表 | 排行榜 |

### Q20: 缓存穿透、击穿、雪崩？

**参考答案**：

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 穿透 | 查询不存在的数据 | 布隆过滤器、缓存空值 |
| 击穿 | 热点 key 过期 | 永不过期、互斥锁 |
| 雪崩 | 大量 key 同时过期 | 随机过期时间、多级缓存 |

**追问**：
- 布隆过滤器原理？
- 如何设计多级缓存？
