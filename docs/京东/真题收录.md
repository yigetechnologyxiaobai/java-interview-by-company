### Q1: SpringMVC 执行流程？

**参考答案**：

```
请求 → DispatcherServlet
     → HandlerMapping（找到 Controller）
     → HandlerAdapter（执行 Controller）
     → Controller（处理业务）
     → ModelAndView（返回视图和数据）
     → ViewResolver（解析视图）
     → View（渲染页面）
     → 响应
```

### Q2: JSP 和 Servlet 的区别？

**参考答案**：

| 特性 | Servlet | JSP |
|------|---------|-----|
| 本质 | Java 类 | 文本文件（编译成 Servlet） |
| 代码风格 | Java 代码为主 | HTML 为主 |
| 角色 | 业务逻辑 | 页面展示 |
| 修改 | 需要重新编译 | 修改后自动编译 |

**MVC 模式**：
- **Model**：JavaBean / DTO
- **View**：JSP
- **Controller**：Servlet

### Q3: 索引可能产生的问题及优化？

**参考答案**：

**问题**：
1. 占用存储空间
2. 降低写操作性能（INSERT/UPDATE/DELETE 需要维护索引）
3. 索引失效导致查询变慢

**优化**：
1. 选择合适的索引类型（B+ 树 vs 哈希）
2. 遵循最左前缀原则
3. 避免索引失效场景
4. 定期分析和优化索引

### Q4: String 字符串池的对象？

**参考答案**：

**字符串池位置**：
- JDK 6：方法区（永久代）
- JDK 7+：堆

**创建方式**：
```java
// 字面量：存入字符串池
String s1 = "hello";

// new：堆中创建新对象
String s2 = new String("hello");

// intern()：如果池中有则返回，否则加入池
String s3 = s2.intern();

System.out.println(s1 == s2);  // false
System.out.println(s1 == s3);  // true
```

### Q5: 手写快排和归并排序？

**快排**：
```java
void quickSort(int[] arr, int left, int right) {
    if (left >= right) return;
    int pivot = partition(arr, left, right);
    quickSort(arr, left, pivot - 1);
    quickSort(arr, pivot + 1, right);
}

int partition(int[] arr, int left, int right) {
    int pivot = arr[right];
    int i = left;
    for (int j = left; j < right; j++) {
        if (arr[j] < pivot) {
            swap(arr, i++, j);
        }
    }
    swap(arr, i, right);
    return i;
}
```

**归并排序**：
```java
void mergeSort(int[] arr, int left, int right) {
    if (left >= right) return;
    int mid = left + (right - left) / 2;
    mergeSort(arr, left, mid);
    mergeSort(arr, mid + 1, right);
    merge(arr, left, mid, right);
}
```

### Q6: 实现一个线程池保证多个线程顺序执行？

**参考答案**：

```java
// 方案 1：单线程线程池
ExecutorService executor = Executors.newSingleThreadExecutor();
for (int i = 0; i < 10; i++) {
    executor.execute(() -> {
        // 任务按顺序执行
    });
}

// 方案 2：使用 join()
Thread t1 = new Thread(() -> {});
Thread t2 = new Thread(() -> {});
t1.start();
t1.join();  // 等待 t1 完成
t2.start();

// 方案 3：使用 CountDownLatch

// 线程 1 执行完 latch.countDown()
```
