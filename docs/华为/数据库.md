## 三、MySQL 高频题


### Q11: MySQL 为什么用 B+ 树？

**参考答案**：

**B+ 树特点**：
1. **树高度低**：3-4 层可存千万级数据
2. **范围查询友好**：叶子节点链表
3. **磁盘利用率高**：非叶子节点只存键值

**与 B 树区别**：
| 特性 | B 树 | B+ 树 |
|------|------|-------|
| 数据存储 | 所有节点 | 只有叶子节点 |
| 范围查询 | 需要中序遍历 | 叶子节点链表 |
| 树高度 | 较高 | 较低 |

### Q12: 索引优化？

**参考答案**：

**索引设计原则**：
1. 选择区分度高的列
2. 遵循最左前缀原则
3. 避免索引列做运算
4. 控制索引数量

**索引失效场景**：
- LIKE 以 % 开头
- 对索引列做运算
- 类型隐式转换
- 使用函数
- OR 条件（部分列无索引）

**EXPLAIN 分析**：
- type：ALL（全表扫描）、index、range、ref、const
- key：实际使用的索引
- rows：预估扫描行数

**追问**：
- 覆盖索引的优势？
- 索引下推是什么？

### Q13: MVCC 原理？

**参考答案**：

**核心组件**：
1. **隐藏字段**：
   - DB_TRX_ID：事务 ID
   - DB_ROLL_PTR：回滚指针

2. **Undo Log**：版本链

3. **Read View**：可见性判断
   - m_ids：活跃事务 ID 列表
   - min_trx_id：最小活跃事务 ID
   - max_trx_id：下一个事务 ID
   - creator_trx_id：创建者事务 ID

**可见性判断**：
- trx_id < min_trx_id → 可见
- trx_id >= max_trx_id → 不可见
- trx_id 在 m_ids 中 → 不可见

**隔离级别**：
- RC：每次 select 生成新的 Read View
- RR：事务第一次 select 生成 Read View

**追问**：
- MVCC 能解决幻读吗？
- 当前读和快照读区别？

### Q14: 主从复制？

**参考答案**：

**复制流程**：
1. Master 写入 Binlog
2. Slave IO 线程读取 Binlog，写入 Relay Log
3. Slave SQL 线程重放 Relay Log

**复制模式**：
- 异步复制：性能好，可能丢数据
- 半同步复制：至少一个从库确认
- 全同步复制：所有从库确认

**主从延迟解决**：
1. 并行复制
2. 读写分离优化（关键读走主库）
3. 分库分表

**追问**：
- 如何监控主从延迟？
- GTID 复制是什么？

## 四、Redis 高频题


### Q15: Redis 为什么快？

**参考答案**：

1. **内存操作**：数据在内存中
2. **单线程**：无锁竞争
3. **IO 多路复用**：epoll
4. **高效数据结构**：SDS、跳表、压缩列表

**追问**：
- 为什么用单线程？
- Redis 6.0 为什么引入多线程？

### Q16: Redis 持久化？

**参考答案**：

| 方式 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| RDB | 快照 | 文件小、恢复快 | 可能丢数据 |
| AOF | 追加命令 | 数据安全 | 文件大、恢复慢 |
| 混合 | RDB + AOF 增量 | 综合 | 复杂 |

**AOF 重写**：
- 触发条件：文件大小超过阈值
- 重写过程：fork 子进程，生成新文件

**追问**：
- RDB 和 AOF 如何选择？
- 混合持久化原理？

### Q17: 缓存穿透、击穿、雪崩？

**参考答案**：

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 穿透 | 查询不存在的数据 | 布隆过滤器、缓存空值 |
| 击穿 | 热点 key 过期 | 永不过期、互斥锁 |
| 雪崩 | 大量 key 同时过期 | 随机过期时间、多级缓存 |

**布隆过滤器**：
- 位图 + 多个哈希函数
- 判断元素可能存在或一定不存在
- 不支持删除

**追问**：
- 布隆过滤器原理？
- 如何设计多级缓存？
