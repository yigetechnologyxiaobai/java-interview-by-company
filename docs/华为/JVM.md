## 二、JVM 高频题


### Q6: JVM 内存结构？

**参考答案**：

**运行时数据区**：
```
┌─────────────────────────────────────┐
│            方法区（元空间）            │
├─────────────────────────────────────┤
│              堆                      │
├──────────┬──────────┬───────────────┤
│ 虚拟机栈  │ 本地方法栈│  程序计数器    │
├──────────┴──────────┴───────────────┤
│            本地内存                   │
└─────────────────────────────────────┘
```

**堆内存划分**：
- 新生代（1/3）：Eden + Survivor0 + Survivor1
- 老年代（2/3）

**方法区变化**：
- JDK 7：永久代（PermGen）
- JDK 8：元空间（Metaspace，本地内存）

**追问**：
- 堆和栈的区别？
- 对象一定在堆上分配吗？

### Q7: 垃圾收集算法？

**参考答案**：

| 算法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 标记-清除 | 标记后清除 | 简单 | 碎片 |
| 标记-整理 | 标记后整理 | 无碎片 | 移动成本 |
| 复制算法 | 复制存活对象 | 高效 | 空间浪费 |
| 分代收集 | 按分代选算法 | 综合 | 复杂 |

**分代收集**：
- 新生代：Eden 区存活率低，用复制算法
- 老年代：存活率高，用标记-整理

**追问**：
- 为什么新生代用复制算法？
- 老年代为什么不用复制算法？

### Q8: CMS 收集器？

**参考答案**：

**四个阶段**：
1. **初始标记**（STW）：标记 GC Roots 直接关联的对象
2. **并发标记**：遍历对象图
3. **重新标记**（STW）：修正并发标记期间的变动
4. **并发清除**：清除垃圾对象

**优点**：
- 低延迟
- 并发收集

**缺点**：
- CPU 敏感（占用线程）
- 浮动垃圾
- 空间碎片

**追问**：
- CMS 和 G1 的区别？
- 为什么 CMS 要用标记-清除？

### Q9: G1 收集器？

**参考答案**：

**核心概念**：
- **Region**：堆划分为多个大小相等的 Region
- **Remembered Set**：跨 Region 引用
- **SATB**：原始快照算法

**回收步骤**：
1. 初始标记（STW）
2. 并发标记
3. 最终标记（STW）
4. 筛选回收（STW）

**特点**：
- 可预测停顿
- 无碎片
- 大内存友好

**追问**：
- G1 如何实现可预测停顿？
- G1 和 CMS 如何选择？

### Q10: 类加载机制？

**参考答案**：

**加载过程**：
1. **加载**：读取 Class 文件，创建 Class 对象
2. **验证**：确保 Class 文件合法
3. **准备**：为静态变量分配内存，设初始值
4. **解析**：符号引用转直接引用
5. **初始化**：执行 `<clinit>` 方法

**双亲委派模型**：
```
启动类加载器
    ↑
扩展类加载器
    ↑
应用类加载器
```

**为什么要双亲委派**：
- 安全性：防止核心类被篡改
- 唯一性：避免类重复加载

**打破双亲委派**：
- 自定义类加载器
- SPI 机制（线程上下文类加载器）
- Tomcat 类加载器

**追问**：
- Tomcat 如何打破双亲委派？
- SPI 机制原理？
