# 阿里巴巴 Java 面试题

## 面试风格

**特点**：深挖底层原理，连环追问，注重思考深度

**轮次**：通常 3-4 轮技术面 + 1 轮 HR

**重点**：JVM、并发、分布式、MySQL、Spring

---

## 一、JVM 高频题

### Q1: 说一下 JVM 内存结构？

**参考答案**：

JVM 运行时数据区分为：

1. **程序计数器**：当前线程执行的字节码行号
2. **虚拟机栈**：方法调用的栈帧（局部变量表、操作数栈）
3. **本地方法栈**：Native 方法服务
4. **堆**：对象实例存储，GC 主要区域
5. **方法区**：类信息、常量、静态变量（JDK 8 后为元空间）

**追问**：
- 堆和栈的区别？
- 方法区在不同 JDK 版本的变化？
- 对象一定在堆上分配吗？（逃逸分析、栈上分配）

### Q2: 垃圾收集算法有哪些？

**参考答案**：

1. **标记-清除**：效率低，有碎片
2. **标记-整理**：无碎片，但移动成本高
3. **复制算法**：高效，但空间利用率低
4. **分代收集**：新生代用复制，老年代用标记-整理

**追问**：
- 为什么新生代用复制算法？（存活率低）
- CMS 用什么算法？有什么问题？（标记-清除，碎片）
- G1 的 Region 设计？

### Q3: 说一下 CMS 收集器？

**参考答案**：

四个阶段：
1. **初始标记**（STW）：标记 GC Roots 直接关联的对象
2. **并发标记**：遍历对象图
3. **重新标记**（STW）：修正并发标记期间的变动
4. **并发清除**：清除垃圾对象

**问题**：
- CPU 敏感（并发占用）
- 浮动垃圾
- 空间碎片

**追问**：
- CMS 和 G1 的区别？
- 为什么 CMS 要用标记-清除不用标记-整理？

### Q4: 类加载过程？

**参考答案**：

1. **加载**：读取 Class 文件，创建 Class 对象
2. **验证**：确保 Class 文件合法
3. **准备**：为静态变量分配内存，设初始值
4. **解析**：符号引用转直接引用
5. **初始化**：执行 `<clinit>` 方法

**追问**：
- 双亲委派模型是什么？为什么要这样设计？
- 如何打破双亲委派？（自定义类加载器、SPI）

## 二、并发编程高频题

### Q5: synchronized 底层原理？

**参考答案**：

- **同步代码块**：monitorenter/monitorexit 字节码指令
- **同步方法**：ACC_SYNCHRONIZED 标志

**锁升级过程**：
1. 无锁 → 偏向锁（同一线程重入）
2. 偏向锁 → 轻量级锁（CAS 自旋）
3. 轻量级锁 → 重量级锁（阻塞等待）

**追问**：
- 锁升级为什么不可逆？
- 偏向锁为什么有延迟？（JVM 启动时竞争多）
- synchronized 和 ReentrantLock 区别？

### Q6: ThreadLocal 原理和内存泄漏？

**参考答案**：

**原理**：
- 每个 Thread 有一个 ThreadLocalMap
- key 是 ThreadLocal（弱引用），value 是值（强引用）

**内存泄漏**：
- key 被 GC 后变 null
- value 仍被 Entry 引用
- 线程不结束则 value 无法回收

**解决**：手动 remove()

**追问**：
- key 为什么用弱引用？
- ThreadLocal 应用场景？（数据库连接、用户上下文）

### Q7: 线程池参数和执行流程？

**参考答案**：

**参数**：
- corePoolSize：核心线程数
- maximumPoolSize：最大线程数
- keepAliveTime：空闲线程存活时间
- workQueue：工作队列
- handler：拒绝策略

**执行流程**：
1. 线程数 < corePoolSize → 创建核心线程
2. 线程数 >= corePoolSize → 放入队列
3. 队列满 → 创建非核心线程
4. 线程数 = maximumPoolSize → 执行拒绝策略

**追问**：
- 为什么不用 Executors 创建线程池？（OOM）
- 拒绝策略有哪些？
- 如何合理设置参数？

### Q8: ConcurrentHashMap 实现？

**参考答案**：

**JDK 1.7**：分段锁（Segment + ReentrantLock）

**JDK 1.8**：CAS + synchronized

**put 流程**：
1. 计算 hash，定位桶
2. 桶为空 → CAS 插入
3. 桶不为空 → synchronized 锁头节点
4. 链表或红黑树插入

**追问**：
- JDK 1.7 和 1.8 的区别？
- 为什么 JDK 1.8 放弃分段锁？

## 三、MySQL 高频题

### Q9: MySQL 为什么用 B+ 树？

**参考答案**：

1. **树高度低**：3-4 层可存千万级数据，IO 次数少
2. **范围查询友好**：叶子节点链表
3. **磁盘利用率高**：非叶子节点只存键值

**追问**：
- B 树和 B+ 树的区别？
- 为什么不用 Hash 索引？

### Q10: 索引失效场景？

**参考答案**：

1. LIKE 以 % 开头
2. 对索引列做运算
3. 类型隐式转换
4. OR 条件（部分列无索引）
5. 使用函数
6. 不等于（!=、<>）
7. IS NOT NULL

**追问**：
- 最左前缀原则？
- 什么是覆盖索引？

### Q11: MVCC 原理？

**参考答案**：

**核心组件**：
- **隐藏字段**：DB_TRX_ID（事务ID）、DB_ROLL_PTR（回滚指针）
- **Undo Log**：版本链
- **Read View**：可见性判断

**隔离级别**：
- RC：每次 select 生成新的 Read View
- RR：事务第一次 select 生成 Read View

**追问**：
- MVCC 能解决幻读吗？（不能完全解决，需要间隙锁）
- 当前读和快照读的区别？

### Q12: 主从复制原理？

**参考答案**：

1. Master 写入 Binlog
2. Slave 的 IO 线程读取 Binlog，写入 Relay Log
3. Slave 的 SQL 线程重放 Relay Log

**主从延迟问题**：
- 原因：单线程重放、大事务
- 解决：并行复制、读写分离优化

**追问**：
- 如何保证主从一致性？
- 半同步复制？

## 四、分布式高频题

### Q13: 分布式事务解决方案？

**参考答案**：

1. **2PC**：准备 + 提交，阻塞、单点问题
2. **TCC**：Try-Confirm-Cancel，业务侵入
3. **本地消息表**：最终一致性
4. **事务消息**：RocketMQ 事务消息
5. **Seata**：AT/TCC/SAGA 模式

**追问**：
- TCC 和 2PC 的区别？
- RocketMQ 事务消息原理？

### Q14: 分布式锁实现？

**参考答案**：

**Redis 实现**：
```java
SET lock:key value NX PX 30000
```

**Zookeeper 实现**：
- 临时顺序节点
- EPHEMERAL_SEQUENTIAL

**对比**：
- Redis：性能高，但可靠性稍差
- Zookeeper：可靠性高，但性能稍差

**追问**：
- Redisson 的看门狗机制？
- Redlock 算法？

### Q15: 如何设计一个秒杀系统？

**参考答案**：

1. **前端**：限流、按钮置灰
2. **网关层**：限流、黑名单
3. **应用层**：Redis 预扣库存
4. **数据层**：MQ 异步下单

**核心设计**：
- 库存预热到 Redis
- 原子扣减（Lua 脚本）
- MQ 异步处理
- 防超卖、防重复

**追问**：
- 如何防止超卖？
- 如何处理高并发下的订单重复？

## 五、Spring 高频题

### Q16: Spring Bean 生命周期？

**参考答案**：

1. **实例化**：创建对象
2. **属性赋值**：依赖注入
3. **初始化**：
   - Aware 接口
   - BeanPostProcessor（Before）
   - InitializingBean
   - init-method
   - BeanPostProcessor（After）
4. **销毁**：DisposableBean、destroy-method

**追问**：
- 循环依赖如何解决？（三级缓存）
- Bean 的作用域？

### Q17: Spring AOP 原理？

**参考答案**：

**动态代理**：
- JDK 动态代理：接口
- CGLIB：类

**核心概念**：
- 切面（Aspect）
- 切点（Pointcut）
- 通知（Advice）
- 连接点（JoinPoint）

**追问**：
- AOP 如何实现事务？
- 多个 AOP 的执行顺序？

### Q18: Spring 事务传播机制？

**参考答案**：

| 传播行为 | 说明 |
|---------|------|
| REQUIRED | 有则加入，无则新建（默认） |
| SUPPORTS | 有则加入，无则非事务 |
| MANDATORY | 必须有事务 |
| REQUIRES_NEW | 新建事务，挂起当前 |
| NOT_SUPPORTED | 非事务执行 |
| NEVER | 非事务执行，有则异常 |
| NESTED | 嵌套事务 |

**追问**：
- REQUIRED 和 NESTED 区别？
- 事务失效场景？
