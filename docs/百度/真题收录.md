#### Q1: RabbitMQ vs Kafka vs RocketMQ 怎么选？

**参考答案**：

| 特性 | RabbitMQ | Kafka | RocketMQ |
|------|----------|-------|----------|
| 吞吐量 | 万级 | 十万级 | 十万级 |
| 延迟 | 微秒级 | 毫秒级 | 毫秒级 |
| 可靠性 | 高 | 一般 | 高 |
| 顺序性 | 不保证 | 分区内有序 | 有序 |
| 事务 | 支持 | 支持 | 支持 |

**选型建议**：
- **RabbitMQ**：业务简单、可靠性要求高、延迟敏感
- **Kafka**：大数据、日志采集、高吞吐
- **RocketMQ**：金融、电商、事务消息

#### Q2: RAG 是什么？文本向量化的难点？

**参考答案**：

**RAG（Retrieval-Augmented Generation）**：
- 检索增强生成，结合向量检索和大模型
- 流程：文档切分 → 向量化 → 存储到向量库 → 查询检索 → 生成回答

**文本向量化难点**：
1. **切分粒度**：太粗丢失精度，太细丢失上下文
2. **多模态**：图文混排的处理
3. **更新增量**：文档更新后如何同步
4. **语义鸿沟**：词向量不一定反映真实语义

#### Q3: Caffeine 多节点缓存一致性？

**参考答案**：

**问题**：本地缓存各节点独立，数据不一致。

**解决方案**：

1. **短 TTL**：过期自动刷新
2. **主动失效**：Redis Pub/Sub 通知
   ```java
   // 写入时发布失效消息
   redis.publish("cache:invalidate:" + key, "1");
   
   // 各节点订阅并清除本地缓存
   redis.subscribe("cache:invalidate:*", (channel, msg) -> {
       caffeineCache.invalidate(extractKey(channel));
   });
   ```
3. **双写一致性**：先更新数据库，再删除缓存

#### Q4: 进程的内存布局？

**参考答案**：

```
高地址
┌─────────────────────┐
│      内核空间        │  用户态不可访问
├─────────────────────┤
│       栈 ↓          │  函数调用、局部变量
├─────────────────────┤
│        ...          │
├─────────────────────┤
│       堆 ↑          │  动态分配（malloc/new）
├─────────────────────┤
│    BSS 段           │  未初始化全局变量
├─────────────────────┤
│    数据段           │  已初始化全局变量
├─────────────────────┤
│    代码段           │  程序指令（只读）
└─────────────────────┘
低地址
```

#### Q5: 算法题 - 不重叠的滑动窗口

**题目**：找出数组中不重叠的滑动窗口的最大数量。

**参考解答**：
```java
public int maxNonOverlap(int[] nums, int k) {
    int count = 0;
    int lastEnd = -1;
    
    for (int i = 0; i <= nums.length - k; i++) {
        if (i > lastEnd) {
            // 检查窗口是否满足条件
            boolean valid = true;
            for (int j = i; j < i + k - 1; j++) {
                if (nums[j] >= nums[j + 1]) {
                    valid = false;
                    break;
                }
            }
            if (valid) {
                count++;
                lastEnd = i + k - 1;
            }
        }
    }
    
    return count;
}
```
