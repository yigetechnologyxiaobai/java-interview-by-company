### 一面

#### 1. HashMap 数据结构组成

**答案**：
- JDK7：数组 + 链表
- JDK8：数组 + 链表 + 红黑树
- 负载因子 0.75，扩容 2 倍

---

#### 2. HashMap 是线程安全的吗？

**答案**：不安全，多线程并发可能导致数据丢失或死循环（JDK7）

---

#### 3. ConcurrentHashMap 是线程安全的吗？

**答案**：是的
- JDK7：分段锁（Segment）
- JDK8：CAS + synchronized 锁单个节点

---

#### 4. CAS 操作

**答案**：
- Compare And Swap，比较并交换
- 原子操作，无需加锁
- ABA 问题：用版本号解决

---

#### 5. 如何解决 hash 冲突？

**答案**：
- 链地址法（HashMap）
- 开放地址法（ThreadLocalMap）
- 再哈希法

---

#### 6. JMM 执行顺序原理

**答案**：
- Java 内存模型，规范多线程可见性
- happens-before 规则保证有序性

---

#### 7. volatile 如何保证有序性？其他线程能见到吗？

**答案**：
- 内存屏障禁止重排序
- 其他线程立即可见（MESI 协议）

---

#### 8. synchronized 和 ReentrantLock 区别

**答案**：
| 特性 | synchronized | ReentrantLock |
|------|-------------|---------------|
| 锁类型 | 隐式锁 | 显式锁 |
| 释放 | 自动释放 | 手动 unlock |
| 公平性 | 非公平 | 可选公平/非公平 |
| 条件 | 单条件 | 多 Condition |
| 实现 | JVM 内置 | AQS |

---

#### 9. 用户输入网址到服务器响应的完整流程

**答案**：
1. DNS 解析域名 → IP
2. TCP 三次握手建立连接
3. TLS 握手（HTTPS）
4. 发送 HTTP 请求
5. 服务器处理请求（反向代理 → 应用服务器）
6. 返回 HTTP 响应
7. 浏览器渲染页面

---

#### 10. TLS 如何加密？

**答案**：
- 对称加密（AES）传输数据
- 非对称加密（RSA/ECDHE）交换密钥
- CA 证书验证身份

---

#### 11. TCP 三次握手传递的包

**答案**：
- SYN（客户端 → 服务端）
- SYN+ACK（服务端 → 客户端）
- ACK（客户端 → 服务端）

---

#### 12. 四次挥手能不能是三次？为什么？

**答案**：
- 不能。服务端可能还有数据要发送
- FIN + ACK 合并只能是服务端没有数据时
- 一般需要四次：FIN → ACK → FIN → ACK

---

#### 13. TCP/IP 各层协议

**答案**：
| 层 | 协议 |
|----|------|
| 应用层 | HTTP, FTP, DNS, SMTP |
| 传输层 | TCP, UDP |
| 网络层 | IP, ICMP |
| 数据链路层 | Ethernet, ARP |
| 物理层 | 物理介质 |

**MTU**：最大传输单元，通常 1500 字节（Ethernet）

---

#### 14. TCP 滑动窗口机制

**答案**：
- 流量控制，防止发送方淹没接收方
- 接收方通告窗口大小
- 发送方根据窗口控制发送速率

---

#### 15. 双亲委派机制原理

**答案**：
- 类加载器委托父类加载器先加载
- Bootstrap → Extension → Application
- 保证类唯一性，避免重复加载

---

#### 16. 线程池核心参数及设置

**答案**：
```java
ThreadPoolExecutor(
    corePoolSize,     // 核心线程数
    maximumPoolSize,  // 最大线程数
    keepAliveTime,    // 空闲超时
    unit,             // 时间单位
    workQueue,        // 任务队列
    threadFactory,    // 线程工厂
    handler           // 拒绝策略
)
```

**设置建议**：
- CPU 密集型：核心线程 = CPU 核数
- IO 密集型：核心线程 = CPU 核数 × 2

---

#### 17. 为什么核心线程数不能超过 CPU 核数？

**答案**：
- CPU 密集型任务，线程数过多会导致线程切换开销
- IO 密集型可以超过，因为线程等待 IO 时不占用 CPU

| ZSet | 排行榜、延时队列 |

#### 19. Redis 宕机后的措施

**答案**：

- AOF 持久化（日志）
- 主从复制 + 哨兵高可用

#### 20. 向量数据库原理

**答案**：
- 存储高维向量（embedding）
- 相似度搜索：余弦相似度、欧氏距离

### 手撕算法

#### 1. 双重检验锁单例模式

```java
public class Singleton {
    private volatile static Singleton instance;
    
    private Singleton() {}
    
    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}
```

**追问**：为什么需要 volatile？

**答案**：防止指令重排序，避免返回未初始化的对象

---

#### 2. 颜色工厂实例（设计模式）

**思路**：工厂模式 + 单例
```java
public interface Color { void fill(); }
public class Red implements Color { public void fill() { System.out.println("Red"); } }
public class Blue implements Color { public void fill() { System.out.println("Blue"); } }

public class ColorFactory {
    public Color getColor(String type) {
        if ("RED".equalsIgnoreCase(type)) return new Red();
        if ("BLUE".equalsIgnoreCase(type)) return new Blue();
        return null;
    }
}
```

#### 7. 算法：从 N 个数中找出最大的 k 个数

**答案**：
```java
// 方法一：堆排序（推荐）
public int[] topK(int[] nums, int k) {
    PriorityQueue<Integer> minHeap = new PriorityQueue<>();
    for (int num : nums) {
        if (minHeap.size() < k) minHeap.offer(num);
        else if (num > minHeap.peek()) {
            minHeap.poll();
            minHeap.offer(num);
        }
    }
    return minHeap.stream().mapToInt(i -> i).toArray();
}
// 方法二：快速选择（O(n)）
```

## 小红书校招 Java 面经（小林coding）（2026-04-03）

### OS

#### 1. IO 多路复用详解

**答案**：
- **概念**：一个进程处理多个 IO 流（时分复用）
- **select/poll**：
  - 遍历所有文件描述符（O(n)）
  - 用户态和内核态两次拷贝
  - select 最大 1024 个 fd
- **epoll**：
  - 红黑树存储所有 fd（O(logn)）
  - 事件驱动，只返回活跃 fd
  - 无上限，解决 C10K 问题

---

#### 2. epoll 边缘触发 vs 水平触发

**答案**：
- **水平触发（LT）**：只要可读就持续通知
- **边缘触发（ET）**：只在状态变化时通知一次
- ET 需要一次性读完数据，配合非阻塞 IO
- ET 效率更高（减少 epoll_wait 调用）

---

### Redis

#### 1. Redis 怎么实现 IO 多路复用？

**答案**：
- 单线程 + epoll 监听多个 socket
- 文件事件处理器处理 IO 事件
- Reactor 模式实现
- 每个请求处理时间 < 1ms，可处理上千请求/秒

---

#### 2. Redis 网络模型是怎样的？

**答案**：
- **Redis 6.0 前**：单 Reactor 单线程
  - 优点：简单、无锁竞争
  - 缺点：无法利用多核 CPU
- **Redis 6.0 后**：网络 IO 多线程
  - 命令执行仍单线程
  - 网络 IO 并行处理
  - 默认只多线程写响应

---

#### 3. Redis 为什么快？

**答案**：
- 内存操作，无磁盘 IO 瓶颈
- 单线程无锁竞争、无上下文切换
- IO 多路复用处理大量连接
- 高效数据结构（跳表、压缩列表）

---

#### 4. Redis 哪些地方用了多线程？

**答案**：
- **后台线程**：
  - bio_close_file：关闭文件
  - bio_aof_fsync：AOF 刷盘
  - bio_lazy_free：异步释放内存
- **Redis 6.0 IO 线程**：
  - io_thd_1/2/3：网络 IO 处理
  - 默认只处理写响应
  - 配置 io-threads-do-reads=yes 开启读

---

#### 5. Redis 跳表实现原理

**答案**：
- ZSET 底层：跳表 + 哈希表
- 多层链表结构：
  - L0 层：所有节点
  - L1、L2 层：跳跃节点（索引）
- 查找复杂度 O(logn)
- 创建节点时随机层数（概率 25%）

---

#### 6. Redis 压缩列表实现原理

**答案**：
- 连续内存块，类似数组
- 表头字段：zlbytes、zltail、zllen、zlend
- 节点结构：prevlen、encoding、data
- 缺点：连锁更新（prevlen 变化导致后续节点更新）
- 替代方案：quicklist、listpack

---

#### 7. 如何保证缓存一致性？

**答案**：
- 缓存属于 AP，不保证强一致性
- 策略：先更新数据库，再删缓存
- 异常处理：
  - 消息队列重试删除
  - Canal 订阅 binlog 删除缓存
- 设置过期时间兜底

---
