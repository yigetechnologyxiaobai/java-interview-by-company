### 一面真题

#### Q1: HashMap 扩容过程，怎么解决哈希冲突？

**参考答案**：

**扩容触发条件**：元素数量 > capacity * loadFactor

**扩容过程**：
1. 创建新数组（容量 2 倍）
2. 重新计算元素位置
3. JDK 8 优化：元素要么留在原位置，要么移到原位置+原容量的位置

**哈希冲突解决**：
- **链地址法**：冲突元素形成链表
- **红黑树转换**：链表长度 ≥ 8 且数组长度 ≥ 64 时转为红黑树

#### Q2: MySQL 慢 SQL 优化思路？

**参考答案**：

**优化流程**：
1. EXPLAIN 分析执行计划
2. 检查索引使用情况
3. 分析扫描行数
4. 避免 SELECT *，使用覆盖索引
5. 分页优化（避免深翻页）
6. 数据量大时考虑分表

---

#### Q3: APM 监控原理？Java Agent 了解过吗？

**参考答案**：

**APM 原理**：
- 通过 Java Agent 在类加载时插桩
- 收集方法执行时间、调用链
- 上报到监控平台

**Java Agent**：
- 使用 Instrumentation API
- 在类加载前或运行时修改字节码
- 代表产品：SkyWalking、Pinpoint

---

#### Q4: G1 最大特点？标记位图、卡表卡页、SATB？

**参考答案**：

**G1 特点**：
- Region 分区，可预测停顿时间
- 无内存碎片

**关键概念**：
- **标记位图**：记录 Region 中哪些对象被标记
- **卡表卡页**：跨 Region 引用标记
- **SATB**：Snapshot-At-The-Beginning，解决并发标记漏标问题

---

#### Q5: TiDB 是什么？和 MySQL 比优缺点？

**参考答案**：

**TiDB**：分布式 NewSQL 数据库

| 对比项 | TiDB | MySQL |
|--------|------|-------|
| 分布式 | 支持 | 不支持 |
| 扩容 | 无限扩展 | 单机限制 |
| HTAP | 支持 | 不支持 |
| 兼容性 | MySQL 协议 | 原生 |

---

#### Q6: 分布式事务几种实现方式？

**参考答案**：

| 方式 | 说明 |
|------|------|
| 2PC/TCC | 强一致性，复杂 |
| 本地消息表 | 最终一致性 |
| 最大努力通知 | 适合通知类 |
| Seata AT | 自动补偿 |

---

#### Q7: Kafka 四个选举？

**参考答案**：

1. **Controller 选举**：ZK 中 /controller 节点竞争
2. **Partition Leader 选举**：ISR 中选择
3. **GroupCoordinator 选举**：消费组协调器选举
4. **消费组 Leader 选举**：组内第一个加入的消费者

---

#### Q8: 如何设计 RPC 框架？

**参考答案**：

**核心组件**：
1. 动态代理（生成客户端调用对象）
2. 序列化（JSON/Hessian/Protobuf）
3. 网络传输（Netty）
4. 服务注册发现（ZK/Nacos）
5. 负载均衡（轮询/随机/权重）
6. 熔断降级
