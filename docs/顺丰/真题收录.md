## 顺丰一面真题（2026-04-15）

**来源**：[帅地玩编程](https://www.iamshuaidi.com/31449.html)
**背景**：社招 Java 开发
**特点**：Java 基础 + JVM + 数据库

### Q1: Set 和 List 的区别是什么？

**参考答案**：

| 特性 | Set | List |
|------|-----|------|
| 元素唯一性 | 不重复 | 可重复 |
| 顺序 | 无序 | 有序 |
| 常用实现 | HashSet | ArrayList |

---

### Q2: Set 如何保证元素不重复？

**参考答案**：

**HashSet 实现**：
- 通过 `hashCode()` 和 `equals()` 判断
- 先计算 hash 值，再比较 equals

**TreeSet 实现**：
- 通过 `compareTo()` 方法判断

---

### Q3: HashMap 的负载因子是什么？

**参考答案**：

**负载因子**：0.75

**原因**：
- 折中时间和空间
- 避免 hash 冲突过多
- 避免频繁扩容

---

### Q4: HashMap 扩容机制是怎样的？

**参考答案**：

**扩容触发**：`size > capacity * loadFactor`

**扩容过程**：
1. 新数组容量翻倍
2. 元素重新计算位置
3. JDK 8 优化：元素在原位置或原位置+原容量

---

### Q5: HashMap 扩容后元素的位置如何变化？

**参考答案**：

**JDK 8 优化**：
- 元素要么在原位置
- 要么在原位置+原容量
- 通过 `(e.hash & oldCap) == 0` 判断

---

### Q6: 线程安全如何实现？

**参考答案**：

**线程安全方案**：
- `synchronized` 关键字
- `ReentrantLock` 锁
- `volatile` 关键字
- `Atomic` 类
- `ConcurrentHashMap` 等并发容器

---

### Q7: synchronized 可以加在哪些地方？

**参考答案**：

**加锁位置**：
1. 实例方法：锁当前对象
2. 静态方法：锁 Class 对象
3. 代码块：锁指定对象

---

### Q8: ReentrantLock 实现原理是什么？

**参考答案**：

**实现原理**：
- 基于 AQS（AbstractQueuedSynchronizer）
- CAS 获取锁
- 失败时加入 CLH 队列等待

---

### Q9: 怎么判断锁是否可重入？

**参考答案**：

**可重入判断**：
- 锁的持有者是否是当前线程
- 通过 Thread ID 判断

---

### Q10: 对象什么时候进入老年代？

**参考答案**：

**晋升老年代条件**：
1. 年龄达到阈值（默认 15）
2. Survivor 空间不足
3. 大对象直接进入老年代

---

### Q11: 线程池参数有哪些？任务提交后的流程是什么？

**参考答案**：

**核心参数**：
- corePoolSize：核心线程数
- maximumPoolSize：最大线程数
- workQueue：阻塞队列
- keepAliveTime：空闲时间

**任务提交流程**：
1. 核心线程未满 → 创建线程执行
2. 核心线程已满 → 加入队列
3. 队列已满 → 创建非核心线程
4. 线程数达最大值 → 执行拒绝策略

---

### Q12: InnoDB 和 MyISAM 的区别是什么？

**参考答案**：

| 特性 | InnoDB | MyISAM |
|------|---------|---------|
| 事务 | 支持 | 不支持 |
| 锁 | 行锁 | 表锁 |
| 外键 | 支持 | 不支持 |
| 索引 | B+树 | B+树 |
| 性能 | 写慢读快 | 写快读慢 |

---

### Q13: 事务的四个隔离级别是什么？

**参考答案**：

| 级别 | 说明 |
|------|------|
| 读未提交 | 可能脏读 |
| 读已提交 | 避免脏读 |
| 可重复读 | 避免脏读、不可重复读 |
| 序列化 | 避免所有问题 |

---

### Q14: 幻读是什么？如何防止？

**参考答案**：

**幻读**：同一事务两次查询结果行数不同

**防止方法**：
- MVCC（快照读）
- Next-Key Lock（当前读）

---

### Q15: 一条 SQL 执行很慢如何分析？

**参考答案**：

**分析步骤**：
1. `EXPLAIN` 查看执行计划
2. 检查索引使用情况
3. 分析扫描行数
4. 查看 `SHOW PROFILE`
5. 检查锁等待情况

## 顺丰 Java 社招面经（2026-04-03）

**来源**：[SegmentFault](https://segmentfault.com/a/1190000042035124)

### 一面真题

#### Q1: HashMap 扩容过程，怎么解决哈希冲突？

**参考答案**：

**扩容触发条件**：元素数量 > capacity * loadFactor

**扩容过程**：
1. 创建新数组（容量 2 倍）
2. 重新计算元素位置
3. JDK 8 优化：元素要么留在原位置，要么移到原位置+原容量的位置

**哈希冲突解决**：
- **链地址法**：冲突元素形成链表
- **红黑树转换**：链表长度 ≥ 8 且数组长度 ≥ 64 时转为红黑树

---

#### Q2: MySQL 慢 SQL 优化思路？

**参考答案**：

**优化流程**：
1. EXPLAIN 分析执行计划
2. 检查索引使用情况
3. 分析扫描行数
4. 避免 SELECT *，使用覆盖索引
5. 分页优化（避免深翻页）
6. 数据量大时考虑分表

---

#### Q3: APM 监控原理？Java Agent 了解过吗？

**参考答案**：

**APM 原理**：
- 通过 Java Agent 在类加载时插桩
- 收集方法执行时间、调用链
- 上报到监控平台

**Java Agent**：
- 使用 Instrumentation API
- 在类加载前或运行时修改字节码
- 代表产品：SkyWalking、Pinpoint

---

#### Q4: G1 最大特点？标记位图、卡表卡页、SATB？

**参考答案**：

**G1 特点**：
- Region 分区，可预测停顿时间
- 无内存碎片

**关键概念**：
- **标记位图**：记录 Region 中哪些对象被标记
- **卡表卡页**：跨 Region 引用标记
- **SATB**：Snapshot-At-The-Beginning，解决并发标记漏标问题

---

#### Q5: TiDB 是什么？和 MySQL 比优缺点？

**参考答案**：

**TiDB**：分布式 NewSQL 数据库

| 对比项 | TiDB | MySQL |
|--------|------|-------|
| 分布式 | 支持 | 不支持 |
| 扩容 | 无限扩展 | 单机限制 |
| HTAP | 支持 | 不支持 |
| 兼容性 | MySQL 协议 | 原生 |

#### Q7: Kafka 四个选举？

**参考答案**：

1. **Controller 选举**：ZK 中 /controller 节点竞争

3. **GroupCoordinator 选举**：消费组协调器选举
4. **消费组 Leader 选举**：组内第一个加入的消费者
