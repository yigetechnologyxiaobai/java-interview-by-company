字节系统设计题很常见

## 三、Java 基础高频题

### Q6: HashMap 扩容机制？

**扩容触发**：size > capacity * loadFactor

**扩容流程**：
1. 容量翻倍
2. 创建新数组
3. 元素重新分配（原位置或原位置 + oldCap）

**追问**：
- 为什么是 2 的幂？（位运算效率高）
- JDK 1.7 死循环问题？

### Q7: ConcurrentHashMap 如何保证线程安全？

**JDK 1.8**：
- CAS + synchronized
- 锁粒度：头节点

### Q8: 线程池核心参数？

```java
ThreadPoolExecutor(
    int corePoolSize,      // 核心线程数
    int maximumPoolSize,   // 最大线程数
    long keepAliveTime,    // 空闲存活时间
    TimeUnit unit,
    BlockingQueue<Runnable> workQueue,
    RejectedExecutionHandler handler
)
```

**执行流程**：
1. 线程数 < core → 创建核心线程
2. 线程数 >= core → 入队
3. 队列满 → 创建非核心线程
4. 线程数 = max → 拒绝策略

## 四、MySQL 高频题

### Q9: 索引为什么用 B+ 树？

1. 树高度低，IO 次数少
2. 叶子节点链表，范围查询友好
3. 非叶子节点只存键值，磁盘利用率高

### Q10: MVCC 原理？

**核心组件**：
- 隐藏字段：DB_TRX_ID、DB_ROLL_PTR
- Undo Log：版本链
- Read View：可见性判断

## 五、Redis 高频题

### Q11: Redis 数据结构？

| 类型 | 底层实现 | 场景 |
|------|---------|------|
| String | SDS | 缓存、计数器 |
| Hash | 哈希表 | 对象 |
| List | 压缩列表 + 双向链表 | 队列 |
| Set | 哈希表 + 整数集合 | 标签 |
| ZSet | 跳表 | 排行榜 |

### Q12: Redis 持久化？

| 方式 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| RDB | 快照 | 文件小、恢复快 | 可能丢数据 |
| AOF | 追加命令 | 数据安全 | 文件大、恢复慢 |

**混合持久化**（Redis 4.0）：RDB + AOF 增量
