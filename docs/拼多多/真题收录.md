### 一面真题

#### Q1: TiDB、RocksDB、LevelDB、LSM 树、SSTable？

**参考答案**：

| 数据库 | 特点 |
|--------|------|
| TiDB | 分布式 NewSQL，底层 TiKV 使用 RocksDB |
| RocksDB | LSM 树存储，高性能 KV |
| LevelDB | RocksDB 原型，Google 开源 |

**LSM 树**：
- 写入快（追加写入）
- 读取慢（多层级合并）
- SSTable：排序字符串表

---

#### Q2: HashMap 位操作？HashSet contains 复杂度？红黑树？

**参考答案**：

**位操作**：
```java
// 计算索引
int index = (n - 1) & hash;
// 高 16 位参与运算
h ^ (h >>> 16)
```

**HashSet contains**：O(1) 平均，O(n) 最差（链表）

**红黑树**：自平衡二叉树，插入删除 O(log n)

---

#### Q3: byte[10KB] 内存分配过程？多大对象进老年代？

**参考答案**：

**分配流程**：
1. TLAB 尝试分配
2. Eden 区分配
3. 大对象直接进老年代

**大对象阈值**：
```
-XX:PretenureSizeThreshold=10KB  // 只对 Serial/ParNew 有效
```

---

#### Q4: TLAB、PLAB、CAS 分配？

**参考答案**：

| 概念 | 说明 |
|------|------|
| TLAB | Thread Local Allocation Buffer，线程私有 Eden 区 |
| PLAB | Promotion LAB，晋升时分配缓冲 |
| CAS | 竞争分配时的原子操作 |

---

#### Q5: 为什么指针压缩？原理？

**参考答案**：

**原因**：
- 64 位 JVM 对象引用占 8 字节
- 开启指针压缩后占 4 字节
- 节省内存

**原理**：
- 堆地址 <= 32GB 时可用 32 位表示
- `-XX:+UseCompressedOops`

---

#### Q6: 类加载过程？类隔离？

**参考答案**：

**加载流程**：加载 → 验证 → 准备 → 解析 → 初始化

**类隔离**：
- 不同 ClassLoader 加载同名类是不同类
- Tomcat 类加载器隔离 WebApp

#### Q7: 多态和重载底层实现？字节码层面？

**参考答案**：

**多态（ invokevirtual ）**：
- 运行时根据对象实际类型查找方法

**重载**：
- 编译时静态分派
- 字节码 invokestatic/invokevirtual 区分

#### Q8: SpringBoot 自动装配原理？SPI？

**参考答案**：

**自动装配**：
1. @EnableAutoConfiguration
2. 扫描 META-INF/spring.factories
3. 按条件注解装配

**SPI**：ServiceLoader 加载接口实现类

#### Q9: Spring 事务实现原理？ACID 如何实现？

**参考答案**：

**实现原理**：
- AOP 代理
- ThreadLocal 绑定连接
- 方法执行前后开启/提交事务

**JDBC 实现事务**：
```java
conn.setAutoCommit(false);
// SQL 操作
conn.commit();  // 或 conn.rollback()
```

---

#### Q10: MySQL 索引 B+树？Page 大小？行溢出？

**参考答案**：

**B+树特点**：
- 非叶子节点不存数据，只存索引
- 叶子节点存数据，双向链表

**Page 大小**：16KB

**行溢出**：
- 单行 > 16KB 时存储到溢出页
- text 最大 65535 字节

---

#### Q11: 系统流量突增 10 倍、100 倍怎么处理？

**参考答案**：

**10 倍**：
- 自动扩容
- 限流熔断

**100 倍**：
- 降级非核心服务
- 异步处理
- 缓存预热

### 手撕算法

#### 单例模式 + volatile

```java
public class Singleton {
    private volatile static Singleton instance;
    
    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}
```

**volatile 作用**：禁止指令重排序，防止返回未初始化对象
