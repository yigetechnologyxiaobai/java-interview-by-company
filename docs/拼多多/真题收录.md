### 二面 & 三面：并发编程大通关

#### Q1: 并发三要素是什么？

**参考答案**：

| 要素 | 含义 | 实现方式 |
|------|------|---------|
| 原子性 | 操作不可分割 | synchronized、Lock、Atomic |
| 可见性 | 修改对其他线程可见 | volatile、synchronized |
| 有序性 | 指令按顺序执行 | volatile、happens-before |

---

#### Q2: 创建线程的三种方式及优缺点？

**参考答案**：

| 方式 | 优点 | 缺点 |
|------|------|------|
| 继承 Thread | 简单直接 | 单继承限制 |
| 实现 Runnable | 灵活、可继承其他类 | 无返回值 |
| 实现 Callable | 有返回值 | 稍复杂 |

---

#### Q3: 线程池拒绝策略有哪些？

**参考答案**：

```java
// 1. AbortPolicy（默认）：抛异常
executor.setRejectedExecutionHandler(new ThreadPoolExecutor.AbortPolicy());

// 2. CallerRunsPolicy：调用者线程执行
executor.setRejectedExecutionHandler(new ThreadPoolExecutor.CallerRunsPolicy());

// 3. DiscardPolicy：直接丢弃
executor.setRejectedExecutionHandler(new ThreadPoolExecutor.DiscardPolicy());

// 4. DiscardOldestPolicy：丢弃最老的任务
executor.setRejectedExecutionHandler(new ThreadPoolExecutor.DiscardOldestPolicy());
```

#### Q4: synchronized vs ReentrantLock 深度对比？

**参考答案**：

| 特性 | synchronized | ReentrantLock |
|------|-------------|---------------|
| 实现 | JVM 关键字 | Java 类 |
| 释放锁 | 自动 | 手动 unlock() |
| 公平性 | 非公平 | 可选公平/非公平 |
| 中断 | 不可中断 | lockInterruptibly() |
| 条件变量 | 单一 | 多 Condition |
| 锁状态 | 无法查询 | isLocked()、isHeldByCurrentThread() |

#### Q5: CAS 的三大问题及解决？

**参考答案**：

| 问题 | 解决方案 |
|------|---------|
| ABA 问题 | AtomicStampedReference（版本号） |
| 循环开销 | 自适应自旋 + 锁升级 |
| 单变量限制 | AtomicReference + CAS 多字段 |

#### Q6: AQS 原理？支持的两种同步方式？

**参考答案**：

**核心组件**：
- `state`：同步状态（volatile int）
- `CLH 队列`：等待线程队列

**两种模式**：
1. **独占模式**：ReentrantLock、ReentrantReadWriteLock.WriteLock
2. **共享模式**：CountDownLatch、Semaphore、ReentrantReadWriteLock.ReadLock

---

#### Q7: CyclicBarrier vs CountDownLatch？

**参考答案**：

| 特性 | CountDownLatch | CyclicBarrier |
|------|---------------|---------------|
| 计数器 | 只能用一次 | 可重置复用 |
| 触发条件 | 计数器减到 0 | 线程数达到阈值 |
| 场景 | 主线程等待子线程 | 线程互相等待 |
| 方法 | countDown() / await() | await() |

---

#### Q8: 死锁如何定位和避免？

**参考答案**：

**定位方法**：
```bash
# 1. jstack 查看线程栈
jstack <pid> | grep -A 10 "deadlock"

# 2. jconsole 可视化工具
# 3. jvisualvm 检测死锁
```

**避免方法**：
1. **破坏循环等待**：按顺序获取锁
2. **破坏请求保持**：一次性获取所有锁
3. **使用 tryLock**：设置超时时间
4. **使用 Lock 接口**：可中断锁

---

#### Q9: Linux 下如何查找占用 CPU 最高的线程？

**参考答案**：

```bash
# 1. 找到占用 CPU 最高的进程
top -H -p <pid>

# 2. 找到占用 CPU 最高的线程
printf "%x" <thread_id>  # 转换为十六进制

# 3. 查看线程栈
jstack <pid> | grep <hex_thread_id> -A 30
```
