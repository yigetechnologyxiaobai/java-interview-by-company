## 四、数据库高频题


### Q8: 数据库事务特点？

**核心概念**：

事务有 ACID 四大特性。

**ACID**：

```
┌─────────────────────────────────────────────────────────────────┐
│                       ACID 特性                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  A - 原子性（Atomicity）                                         │
│     └─ 事务是不可分割的最小单位                                 │
│     └─ 要么全部成功，要么全部失败                               │
│     └─ 实现：Undo Log                                          │
│                                                                 │
│  C - 一致性（Consistency）                                       │
│     └─ 事务执行前后，数据保持一致                               │
│     └─ 约束不被破坏                                             │
│     └─ 是最终目标                                               │
│                                                                 │
│  I - 隔离性（Isolation）                                         │
│     └─ 多个事务并发执行互不干扰                                 │
│     └─ 四种隔离级别                                             │
│     └─ 实现：锁 + MVCC                                         │
│                                                                 │
│  D - 持久性（Durability）                                        │
│     └─ 事务提交后，数据永久保存                                 │
│     └─ 实现：Redo Log                                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Q9: 事务隔离级别？

**核心概念**：

隔离级别解决并发事务的问题。

**隔离级别对比**：

| 隔离级别 | 脏读 | 不可重复读 | 幻读 |
|---------|------|-----------|------|
| READ UNCOMMITTED | ✅ | ✅ | ✅ |
| READ COMMITTED | ❌ | ✅ | ✅ |
| REPEATABLE READ | ❌ | ❌ | ✅ |
| SERIALIZABLE | ❌ | ❌ | ❌ |

**MySQL 默认 RR**，通过 MVCC + 间隙锁解决幻读。
