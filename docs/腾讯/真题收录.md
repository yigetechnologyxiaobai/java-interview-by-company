### 转转二面

#### 1. 算法：最长不重复子串

**题目**：给定字符串 s，找出不含重复字符的最长子串长度

**答案**：滑动窗口
```java
public int lengthOfLongestSubstring(String s) {
    Map<Character, Integer> map = new HashMap<>();
    int max = 0, left = 0;
    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        if (map.containsKey(c)) {
            left = Math.max(left, map.get(c) + 1);
        }
        map.put(c, right);
        max = Math.max(max, right - left + 1);
    }
    return max;
}
```

#### 2. 算法：除自身以外数组的乘积

**答案**：左右乘积数组
```java
public int[] productExceptSelf(int[] nums) {
    int n = nums.length;
    int[] left = new int[n];
    int[] right = new int[n];
    left[0] = 1;
    for (int i = 1; i < n; i++) left[i] = left[i-1] * nums[i-1];
    right[n-1] = 1;
    for (int i = n-2; i >= 0; i--) right[i] = right[i+1] * nums[i+1];
    int[] ans = new int[n];
    for (int i = 0; i < n; i++) ans[i] = left[i] * right[i];
    return ans;
}
```

#### 1. 算法：手撕 LRU

**答案**：
```java
class LRUCache {
    LinkedHashMap<Integer, Integer> map;
    int capacity;
    
    public LRUCache(int capacity) {
        this.capacity = capacity;
        this.map = new LinkedHashMap<>(capacity, 0.75f, true);
    }
    
    public int get(int key) {
        return map.getOrDefault(key, -1);
    }
    
    public void put(int key, int value) {
        map.put(key, value);
        if (map.size() > capacity) {
            map.remove(map.keySet().iterator().next());
        }
    }
}
```

### 操作系统

#### 1. 进程和线程的区别

**答案**：
- **本质区别**：进程是资源分配单位，线程是调度执行单位
- **开销**：进程切换开销大（独立内存），线程切换开销小（共享内存）
- **稳定性**：线程崩溃可能影响整个进程，进程间隔离
- **内存**：进程独立内存空间，线程共享进程资源

---

#### 2. 为什么进程崩溃不会影响其他进程？

**答案**：
- **进程隔离性**：每个进程有独立内存空间
- **进程独立性**：不共享资源（文件、网络连接等）
- 一个进程崩溃后内存被回收，不影响其他进程

---

### 数据结构

#### 1. 排序算法及时间复杂度

**答案**：
| 排序算法 | 最好 | 最坏 | 平均 | 空间 |
|---------|------|------|------|------|
| 冒泡 | O(n) | O(n²) | O(n²) | O(1) |
| 插入 | O(n) | O(n²) | O(n²) | O(1) |
| 选择 | O(n²) | O(n²) | O(n²) | O(1) |
| 快速 | O(nlogn) | O(n²) | O(nlogn) | O(logn) |
| 归并 | O(nlogn) | O(nlogn) | O(nlogn) | O(n) |
| 堆 | O(nlogn) | O(nlogn) | O(nlogn) | O(1) |

---

#### 2. 归并排序 vs 快速排序使用场景

**答案**：
- **归并排序**：稳定排序，适合需要保持元素相对顺序的场景
- **快速排序**：不稳定排序，但平均时间最短，适合随机分布数据

---

#### 3. 排序稳定是什么意思？

**答案**：
- 相同关键字元素排序后相对位置不变
- 例：排序前 A 在 B 前面（值相同），排序后 A 仍在 B 前面

#### 6. MVCC 是怎么实现的？

**答案**：
- **Read View**：活跃事务 ID 集合（min_trx_id、max_trx_id、m_ids）
- **Undo Log**：版本链
- **可见性判断**：
  - trx_id < min_trx_id → 可见（已提交）
  - trx_id ≥ max_trx_id → 不可见（新事务）
  - trx_id 在 m_ids 中 → 不可见（未提交）

### MQ

#### 1. RocketMQ vs Kafka 消息确认机制

**答案**：
- **Kafka acks**：
  - 0：不等待确认（最快，可能丢）
  - 1：等待 Leader 确认
  - -1：等待所有副本确认（最可靠）
- **RocketMQ**：
  - 同步发送：等待响应
  - 异步发送：回调确认
  - 单向发送：不等待（最快）

---

#### 2. Kafka vs RocketMQ Broker 架构

**答案**：
- **Kafka**：分布式 Broker，Topic 分区存储在不同节点
- **RocketMQ**：主从 Broker，主节点写入，从节点复制
